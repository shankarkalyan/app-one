LangGraph Design Patterns Used

1. Supervisor Pattern (Conditional Edge Routing)
The Supervisor Router is not a standalone agent — it is implemented as conditional edges in LangGraph's StateGraph. After each agent node completes, the supervisor inspects the shared State dict and uses Command(goto=...) to route to the next node dynamically. This replaces hard-coded linear flows with intelligent, state-aware routing.
2. Reusable SQ Review Node (Subgraph Pattern)
The SQ Review appears 7 times in the flow. Instead of duplicating logic, it is implemented as a single reusable subgraph that reads state["sq_review_context"] to know which checkpoint it is evaluating. It writes pass/fail back to state, and the supervisor routes accordingly. This follows LangGraph's subgraph-as-node pattern.
3. Human-in-the-Loop (interrupt_before)
At the Underwriting Decision checkpoint, interrupt_before=["decision_node"] pauses the graph execution and surfaces the decision to a human underwriter. The human provides Command(resume={"decision": "yes"/"no"}) to continue the flow. This is also recommended before sending the Denial Letter.
4. Command Pattern for Dynamic Routing
Agents return Command(goto="next_node", update={"key": "value"}) instead of plain state updates. This lets each agent explicitly control where the flow goes next, which is critical for the branching logic (Yes/No decisions, SQ pass/fail loops, document-needed checks).
5. Shared State with TypedDict + Annotated Reducers
All agents read/write to a single LoanState(TypedDict). Fields use Annotated[list, add_messages] reducers for message history and simple overwrites for status fields. This ensures consistency across all nodes without side effects.

Utility Nodes

Node Type Description
SQ Review Node Reusable Subgraph Single implementation invoked 7 times. Reads sq_review_context to know which document to review. Returns pass/fail. Max 3 retries before human escalation.
NotifyAgent Utility Node Centralized notification handler. Sends all letters — ineligible, incomplete, missing docs, commitment, denial, closure. Reads notification_type from state.
CallAgent Task Node Assigns to call agent and facilitates review with applicant.
Human-in-the-Loop Interrupt Pauses graph at underwriting decision. Uses interrupt_before. Human provides Yes/No via Command(resume=...).

SQ Review Node - 7 Checkpoints

# Checkpoint On Pass On Fail
1 Disclosure Package Application assigned for Loan Review Loop back to DisclosureAgent
2 Missing Documents Letter Send Letter to Customer Loop back to DocLetterAgent
3 Commitment Letter Send Letter via NotifyAgent Loop back to CommitmentAgent
4 Closing Packet Send Closing Packet to Title Agency Loop back to ClosingPacketAgent
5 Review Closing Packet Complete System Maintenance Loop back to ReviewClosingAgent
6 System Maintenance MSP END - Loan Closed Loop back to MaintenanceAgent
7 Denial Letter Send Denial Letter via NotifyAgent Loop back to DenialAgent

Loop Guards and Safety

SQ Review Loops: Max 3 retries per checkpoint. After 3 failures, escalate to Human Reviewer via interrupt_before.

Document Collection Loop: Max 2 re-requests for missing docs. After 2 cycles with no return, auto-transition to Customer Withdrawn path.

Underwriter Readiness Loop: Max 3 review cycles. After 3 not-ready results, escalate to senior underwriter.

All loops tracked via sq_retry_count in state, reset to 0 when entering a new checkpoint.

Human-in-the-Loop Checkpoints

Checkpoint Why Implementation
Underwriting Decision Yes/No loan decision requires human judgment interrupt_before=["decision_node"]
Denial Letter Send Regulatory requirement for human approval before denial interrupt_before=["send_denial"]
SQ Escalation (any) After 3 SQ failures, human must intervene interrupt_before=["sq_escalation"]
